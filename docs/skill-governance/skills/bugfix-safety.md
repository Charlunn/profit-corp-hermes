# Skill: bugfix-safety

- 名称：bugfix-safety
- 版本：v0.2
- 维护人：repo owner

## 2. 适用场景
- 线上/测试环境缺陷修复
- 需要控制回归风险的修复任务

## 3. 非适用场景
- 新功能开发（应走 feature-to-pr）
- 仅格式或命名整理

## 4. 输入要求
### 必需输入
- 缺陷现象与期望行为
- 复现步骤
- 影响范围（用户、模块、路径）

### 可选输入
- 错误日志片段
- 关联 issue/工单

### 缺失输入处理
- 无法复现时，先补充观测信息，不进入修复承诺

## 5. 执行步骤
1. 先复现：记录最小复现条件
2. 定位根因：区分业务逻辑错误/状态错误/外部依赖错误
3. 最小修复：仅改必要位置，避免搭车重构
4. 回归验证：验证原问题 + 邻近关键路径
5. 风险登记：标注是否存在同类潜在问题

## 6. 输出契约（Output Contract）
必须输出以下结构：
- 问题复现证据
- 根因说明
- 修复摘要
- 回归验证证据
- 回滚方案

## 7. DoD（完成定义）
- [ ] 具备复现与修复后对比证据
- [ ] 回归验证覆盖主路径与高相邻风险点
- [ ] 修复不引入新的分层违规
- [ ] 失败时可执行回滚

## 8. 反模式与失败案例
- 反模式：无法复现就直接“猜修”
- 风险：问题漂移、重复事故
- 正确替代：先完善日志和复现条件，再做修复

- 对齐条目：测试门禁、日志规范、发布回滚
