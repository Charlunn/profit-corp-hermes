# 发布一页纸检查清单（v0.2）

> 任意任务在合并/发布前必须执行。建议直接复制到 PR 描述中逐项勾选。

## A. 硬门禁（任一不通过即阻断）
- [ ] 分层依赖无违规（app -> domain -> infrastructure）
- [ ] lint / format / type-check 全通过
- [ ] 测试证据齐全（至少主路径 + 失败路径）
- [ ] 日志符合结构化规范（关键路径可追踪）
- [ ] 回滚方案明确且可执行

## B. 软门禁（建议通过，未通过需说明）
- [ ] 命名与目录符合约定
- [ ] 影响面说明完整（UI/API/DB/权限/日志）
- [ ] Runbook 已更新（关键链路建议必须）

## C. 发布记录
- 发布负责人：
- 版本/标识：
- 时间：
- 风险等级（low/med/high）：

## D. 失败处理
- 若硬门禁失败：停止合并/发布，回到修复流程
- 若发布后异常：优先止血（回滚/熔断/降级），再做根因修复
