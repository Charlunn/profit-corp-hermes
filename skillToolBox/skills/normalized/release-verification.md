# Skill: release-verification（中立版）

- 名称：release-verification
- 版本：v0.2
- 来源：Google SRE 发布可靠性实践（重写）

## 适用场景
- 发布前最终验证

## 非适用场景
- 开发中间阶段

## 输入要求
- 待发布变更清单
- 关键链路与监控指标
- 回滚方案

## 执行步骤
1. 运行发布前 smoke（关键用户流程）。
2. 核对监控与告警可用性。
3. 核对配置一致性与迁移状态。
4. 判定可发布性并记录发布窗口建议。

## 输出契约
- 验证结果
- 风险等级
- 发布建议
- 失败止血路径

## DoD
- [ ] 关键路径 smoke 通过
- [ ] 监控和告警就绪
- [ ] 回滚演练路径明确

## 风险与回滚
- 风险：发布后缺乏观测导致延迟发现异常
- 回滚：立即按回滚预案执行并恢复前版本
