# Skill: git-auto-commit-policy（中立版）

- 名称：git-auto-commit-policy
- 版本：v0.2
- 来源：pre-commit, commitlint, 工程门禁实践（重写）

## 适用场景
- 需要 AI 在工作分支自动提交变更

## 非适用场景
- 受保护分支（main/release）
- 门禁未通过（lint/type-check/test 失败）

## 输入要求
- 当前分支名称
- 提交范围（文件列表）
- 门禁结果（lint/type-check/test）

## 执行步骤
1. 校验当前分支是否允许自动提交。
2. 运行门禁检查并阻断失败提交。
3. 生成结构化中文提交信息。
4. 提交后输出变更摘要与回滚指令。

## 输出契约
- 提交结果（成功/失败）
- 提交信息
- 提交哈希
- 回滚命令建议

## DoD
- [ ] 仅在允许分支提交
- [ ] 门禁全部通过
- [ ] 提交信息符合中文规范

## 风险与回滚
- 风险：错误自动提交扩大影响
- 回滚：优先使用 `git revert <hash>`，保留共享历史可追溯性
