# Skill: test-strategy-planner（中立版）

- 名称：test-strategy-planner
- 版本：v0.2
- 来源：Google SWE Book, 测试金字塔实践（重写）

## 适用场景
- 新功能开发前/重构前测试策略设计

## 非适用场景
- 纯文档更新

## 输入要求
- 需求与验收标准
- 变更影响面
- 风险等级

## 执行步骤
1. 按风险分层：低/中/高风险功能点。
2. 选择测试层次：单元/集成/E2E 的最小覆盖组合。
3. 定义主路径与失败路径验证。
4. 输出优先级执行计划（先快后慢）。

## 输出契约
- 风险-测试映射表
- 测试层次与范围
- 执行优先级

## DoD
- [ ] 高风险点有明确验证方案
- [ ] 测试层次选择可解释
- [ ] 覆盖主路径与失败路径

## 风险与回滚
- 风险：过度依赖单一测试层导致漏检
- 回滚：补充缺失层次并重新评估门禁
