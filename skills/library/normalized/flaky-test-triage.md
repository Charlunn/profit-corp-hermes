# Skill: flaky-test-triage（中立版）

- 名称：flaky-test-triage
- 版本：v0.2
- 来源：Google Testing Blog, pytest/Playwright 实践（重写）

## 适用场景
- 测试结果不稳定（偶发失败）

## 非适用场景
- 可稳定复现的确定性失败

## 输入要求
- 失败测试标识
- 失败日志与环境信息
- 最近相关改动

## 执行步骤
1. 重复执行确认是否为 flaky。
2. 分类根因：时序/依赖/环境/数据污染/选择器不稳。
3. 采取措施：隔离、修复、暂时降级门禁（需记录）。
4. 设置 owner 与修复期限。

## 输出契约
- flaky 判定结果
- 根因分类
- 修复计划
- 临时策略与过期时间

## DoD
- [ ] flaky 与 hard fail 区分清楚
- [ ] 有明确 owner 与修复时限
- [ ] 不以永久重试掩盖问题

## 风险与回滚
- 风险：长期容忍 flaky 造成门禁失真
- 回滚：恢复阻断并优先修复不稳定测试
