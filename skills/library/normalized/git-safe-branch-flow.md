# Skill: git-safe-branch-flow（中立版）

- 名称：git-safe-branch-flow
- 版本：v0.2
- 来源：gitworkflows, GitHub 受保护分支实践（重写）

## 适用场景
- 任意开发任务开始前的分支与合并策略配置
- 多人/多 AI 并行协作

## 非适用场景
- 一次性本地实验且不进入主仓库

## 输入要求
- 基线分支（通常 main）
- 任务标识（需求/缺陷号）
- 保护分支策略（是否必须 PR）

## 执行步骤
1. 创建工作分支：一任务一分支。
2. 在工作分支完成变更与验证，不直推受保护分支。
3. 仅在质量门禁通过后发起合并请求。
4. 合并后清理工作分支并记录变更归档。

## 输出契约
- 分支命名与用途
- 合并前门禁状态
- 合并建议（通过/阻断）

## DoD
- [ ] 无直接提交受保护分支
- [ ] 门禁通过后才允许合并
- [ ] 分支生命周期可追踪

## 风险与回滚
- 风险：并发任务互相污染
- 回滚：按工作分支回退，不污染主线历史
