# Skill: observability-check（中立版）

- 名称：observability-check
- 版本：v0.2
- 来源：open-telemetry/opentelemetry-specification（重写）

## 适用场景
- 新功能上线前观测性检查
- 故障复盘后的观测改进

## 非适用场景
- 纯 UI 调整且无关键流程变化

## 输入要求
- 关键业务流程
- 现有日志字段与告警规则

## 执行步骤
1. 定义关键链路与关键事件。
2. 对齐日志字段与 trace/request 关联键。
3. 检查错误码、延迟、失败率可观测性。
4. 验证告警阈值与排查入口。

## 输出契约
- 链路观测覆盖表
- 缺口清单
- 修复建议
- 验证证据

## DoD
- [ ] 关键链路具备可追踪日志
- [ ] 错误与延迟可统计
- [ ] 复盘可定位到模块级

## 风险与回滚
- 风险：日志过多导致噪声
- 回滚：保留核心字段，逐步下调低价值日志
