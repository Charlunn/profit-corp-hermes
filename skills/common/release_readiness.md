# Skill: release-readiness（中立版）

- 名称：release-readiness
- 版本：v0.2
- 来源：microsoft engineering playbook, thoughtbot/guides（重写）

## 适用场景
- 合并前、发布前检查

## 非适用场景
- 尚未完成开发与验证的任务

## 输入要求
- 当前变更清单
- 验证证据
- 回滚方案

## 执行步骤
1. 检查硬门禁（分层/lint/type-check/测试/日志/回滚）。
2. 检查软门禁（命名、影响面、Runbook 更新）。
3. 评估风险等级并标注发布窗口。
4. 确认异常止血路径可执行。

## 输出契约
- 门禁检查结果
- 风险等级
- 发布建议（通过/阻断）
- 失败处理路径

## DoD
- [ ] 所有硬门禁通过
- [ ] 关键风险有预案
- [ ] 发布记录可追溯

## 风险与回滚
- 风险：发布前遗漏依赖检查
- 回滚：执行预设回滚路径，冻结增量变更
