## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authority record → governed wrapper | Untrusted persisted project metadata crosses into credentialed action selection and target resolution |
| governed wrapper → GitHub/Vercel helpers | High-impact credentials are exercised across the boundary to external CLIs and platform APIs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | E | scripts/approved_delivery_governance.py | mitigate | Enforce a closed action allowlist and reject any non-approved credential action before helper dispatch, per D-02. |
| T-12-02 | R | scripts/start_approved_project_delivery.py | mitigate | Persist per-action audit artifacts and append linked authority events for success, blocked, and failure outcomes so credential use is non-repudiable, per D-04 through D-06. |
| T-12-03 | T | scripts/github_delivery_common.py / scripts/vercel_delivery_common.py | mitigate | Preserve explicit block reasons and evidence-path returns from helper boundaries so the authority layer cannot silently rewrite credential failure context. |
</threat_model>
