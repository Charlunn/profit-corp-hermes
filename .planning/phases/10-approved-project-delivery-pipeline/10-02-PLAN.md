<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| approved-project authority → workspace instantiation | top-level machine state triggers filesystem creation and identity injection |
| workspace bootstrap → conformance gate | generated workspace contents are validated before downstream execution |
| Phase 10 bootstrap → Phase 9 delivery runtime | Phase 10 passes approved inputs into the workspace-local run initializer |
| workspace final handoff artifact → approved-project authority layer | workspace-local completion evidence is promoted into top-level operator-facing state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-07 | T | `start_approved_project_delivery.py` stage transitions | mitigate | persist every transition as JSONL events and update latest status from that stream only |
| T-10-08 | D | duplicate bootstrap attempts | mitigate | reject duplicate active bootstrap for the same approved project with an explicit blocked state and resume instructions |
| T-10-09 | T | generated workspace before conformance | mitigate | run `check_template_conformance.py` as the required blocking gate before Phase 9 handoff |
| T-10-10 | R | failed bootstrap evidence loss | mitigate | store block reason, failing stage, workspace path, and evidence links in the top-level record and status view |
| T-10-11 | E | bypassing existing governance/conformance stack | mitigate | call existing instantiation, conformance, and delivery-run helpers rather than custom duplicate logic per D-09 |
| T-10-12 | S | forged resume target | mitigate | derive resume stage from the persisted approved-project record and event stream, not from unchecked CLI freeform input |
| T-10-13 | T | mismatched final handoff links across artifacts | mitigate | write final handoff artifact reference once through the controller, reuse that value for record/event/status updates, and test exact cross-surface equality |
</threat_model>
