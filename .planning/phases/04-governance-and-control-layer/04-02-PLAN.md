<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator/cron request → governance request CLI | 高影响动作首先进入治理分类与审批流 |
| governance wrapper → authoritative writer | 包装器决定是否允许实际状态写入发生 |
| failure path → governance history | 关键步骤失败必须留下 failed/blocked 事件 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05 | E | `request_governance_approval.py` override path | mitigate | `override` 仅允许 CEO；非 CEO 请求直接失败。 |
| T-04-06 | T | governance status transitions | mitigate | 仅允许合法状态机迁移：pending→approved/rejected/override，异常迁移拒绝落盘。 |
| T-04-07 | R | blocked/failed execution | mitigate | 包装器在 exit non-zero 前先写 `blocked` 或 `failed` 事件，避免 silent stop。 |
| T-04-08 | I | high-impact action execution | mitigate | 缺审批、被 reject、或 critical failure 时绝不调用下游 authoritative writer。 |
</threat_model>
