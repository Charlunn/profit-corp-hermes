<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authority GitHub metadata → workspace git remote | Approved repo metadata crosses into live remote mutation and must not drift to the wrong target |
| sync helper → network transport | Push behavior crosses from local workspace commands into external GitHub transport pathways |
| helper evidence → authority persistence | Failure-boundary evidence becomes operator-facing state and must remain specific enough to support blocking decisions |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-04 | T | scripts/github_delivery_common.py remote convergence | mitigate | Converge missing/mismatched remotes from approved authority metadata and lock the behavior with regression tests before any push occurs. |
| T-15-05 | D | scripts/github_delivery_common.py push transport | mitigate | Implement transport-aware push behavior so HTTPS-only failure does not block a healthy operator path; execution must stop and surface explicit evidence if no safe transport succeeds. |
| T-15-06 | R | github sync evidence + controller persistence | mitigate | Persist step-specific failure metadata (`remote`, `branch`, `stage`, `commit`, `push`) so high-severity ambiguity is blocked rather than hidden behind a generic error. |
</threat_model>
