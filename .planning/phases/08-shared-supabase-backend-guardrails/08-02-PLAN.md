<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Generated workspace SQL/client code → Hermes conformance gate | 不受信任的产品代码跨入平台校验器，若扫描规则过弱会放过共享后端越界 |
| Template source protected files → workspace protected files | 平台受保护原语从 canonical template 映射到生成工作区，漂移必须被检测 |
| Client components → shared tables | 浏览器代码若直接写共享支付/权益状态，会绕过预期 server-owned 流程 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-05 | T | `supabase/migrations/*.sql` | mitigate | 在 `check_template_conformance.py` 中扫描 `create table`，仅允许四张共享表无前缀，其余必须匹配当前 `APP_KEY_`。 |
| T-08-06 | T | `src/app/**`, `src/components/**` client files | mitigate | 对 `"use client"` 文件执行共享表 mutation pattern 扫描，阻断 `.insert/.update/.upsert/.delete` 直接写共享支付/权益状态。 |
| T-08-07 | E | alternate backend bootstrap files/config | mitigate | 校验 `.hermes` 护栏元数据、Supabase helper 路径与 contract 一致，阻断默认第二后端模型渗入。 |
| T-08-08 | T | protected auth/payment/entitlement helpers | mitigate | 扩展 protected manifest 覆盖 `src/lib/entitlement.ts`、`src/lib/supabase-browser.ts`、`src/lib/supabase-server.ts`、`src/lib/paypal.ts`，用 sha256 漂移检测拦截共享流程替换。 |
</threat_model>
