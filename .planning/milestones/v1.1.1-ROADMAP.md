# Milestone v1.1.1: Delivery Pipeline Reliability Fixes

**Status:** ✅ SHIPPED 2026-04-30
**Phases:** 14-17
**Total Plans:** 10

## Overview

Profit-Corp Hermes v1.1.1 closes the live-delivery reliability gap exposed after the first shipped SaaS factory milestone. The system now accepts real operator GitHub and Vercel auth paths, syncs real Windows + pnpm workspaces more reliably, and converges authority plus operator-facing status artifacts back to the true recovered final state after successful delivery.

## Milestone Summary

**Decimal Phases:**

- None

**Key Decisions:**
- Decision: Treat an already-authenticated `gh` CLI session as a valid governed GitHub auth source when env tokens are absent
- Decision: Resolve canonical GitHub repository owner/name/url once at the controller layer and persist that shape across bootstrap and resume
- Decision: Use source-only snapshots, remote convergence, and transport fallback to make GitHub sync match the real Windows + pnpm operator environment
- Decision: Accept either explicit `VERCEL_TOKEN` or a healthy local Vercel CLI session while preserving operator-visible auth-source evidence
- Decision: Make recovered authority truth authoritative for current-state rendering while preserving blocked history only as audit evidence
- Decision: Close the milestone against regression and validation evidence instead of redesigning the approved-delivery architecture

**Issues Resolved:**
- Resolved `missing_github_auth` failures when `gh` login was healthy but token env vars were absent
- Resolved repository-owner fallback to project slug during approved-project bootstrap and resume
- Resolved real-workspace GitHub sync failures caused by oversized snapshots, long paths, and unhealthy transport assumptions
- Resolved Vercel token-only gating that ignored valid local CLI login state
- Resolved stale authority/status artifacts that kept recovered successful runs visually blocked after GitHub/Vercel success
- Resolved missing end-to-end regression coverage across helper, controller, bootstrap, resume, governance, and CLI validator seams for the repaired delivery path

**Issues Deferred:**
- A fresh live end-to-end testing round is still needed to validate the repaired delivery path on the real operator machine after milestone close
